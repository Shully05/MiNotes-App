170总结：
171    数据层：通过 NotesProvider 管理 SQLite 数据库。
172    界面层：包含列表、编辑、设置、闹钟弹窗等多个 Activity。
173    后台层：包含同步服务（SyncService）和闹钟管理（AlarmReceiver）。
174    扩展性：支持桌面小部件（Widget）和全局搜索。
175
176    共四个界面组件（Activity）：
177    1. NotesListActivity：主界面，显示笔记列表。
178    2. NoteEditActivity：编辑界面，处理笔记的创建和修改。
179    3. AlarmAlertActivity：闹钟弹窗界面，提醒用户。
180    4. NotesPreferenceActivity：设置界面，允许用户配置应用选项。
181    一个数据提供者（Provider）：
182    NotesProvider：管理笔记数据的存储和访问。
183    四个广播接收器（Receiver）：
184    1. NoteWidgetProvider_2x 和 NoteWidgetProvider_4x：管理桌面小部件的更新和删除。
185    2. AlarmInitReceiver：监听系统启动完成事件，初始化闹钟设置。
186    3. AlarmReceiver：处理闹钟触发事件。
187    一个服务（Service）：
188    GTaskSyncService：负责与 Google Tasks 进行数据同步。
189
190    项目的入口是 NotesListActivity。整个应用的核心交互逻辑围绕着‘列表展示’和‘单条编辑’两个页面展开，
191    同时通过 AlarmAlertActivity 实现了全屏闹钟提醒功能，架构非常清晰。
